# Data Crokinole - Static Site

A DataGolf-inspired statistics portal for competitive Crokinole, featuring FlickSkill rankings, season points, tournament data, and detailed player profiles.

## üöÄ Quick Deploy to GitHub Pages

### Step 1: Create GitHub Repository

1. Go to [GitHub](https://github.com) and create a new repository
2. Name it something like `data-crokinole` or `crokinole-stats`
3. Make it **Public** (required for free GitHub Pages)
4. Don't initialize with README (we'll upload files directly)

### Step 2: Upload Files

**Option A: Via GitHub Web Interface**
1. Click "uploading an existing file"
2. Drag and drop ALL files from this folder:
   - `index.html`
   - `dc_rankings.html`
   - `dc_points.html`
   - `dc_tournaments.html`
   - `styles.css`
   - `player_data.json`
   - All `player_*.html` files
   - All `tournament_*.html` files
3. Commit with message: "Initial Data Crokinole site"

**Option B: Via Git Command Line**
```bash
cd /path/to/this/folder
git init
git add .
git commit -m "Initial Data Crokinole site"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/YOUR-REPO.git
git push -u origin main
```

### Step 3: Enable GitHub Pages

1. Go to repository **Settings** ‚Üí **Pages**
2. Under "Source", select **Deploy from a branch**
3. Choose branch: **main**
4. Choose folder: **/ (root)**
5. Click **Save**

Wait 1-2 minutes, then your site will be live at:
```
https://YOUR-USERNAME.github.io/YOUR-REPO/
```

## üîÑ Updating the Site

### Manual Updates

When you have new tournament data:

1. In your Streamlit app:
   - Upload new tournament CSV
   - Recalculate FlickSkill + Points
   - Go to "Data Crokinole" page
   - Download new ZIP file
   
2. Extract ZIP and upload to GitHub:
   ```bash
   # Replace all files in your repo
   cp /path/to/new/files/* .
   git add .
   git commit -m "Update rankings - Tournament Name YYYY-MM-DD"
   git push
   ```

3. GitHub Pages auto-updates in ~1 minute

### Automated Updates (Optional)

Add this GitHub Action to auto-deploy when files change:

Create `.github/workflows/deploy.yml`:
```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./
```

## üêõ Troubleshooting

**Page shows 404 after deploy:**
- Wait 2-3 minutes for GitHub to build
- Check Settings ‚Üí Pages shows green checkmark
- Verify files are in root folder (not subfolder)

**Charts not loading:**
- Ensure `player_data.json` is uploaded
- Check browser console for errors (F12)

**Broken links:**
- All links use relative paths
- Don't add subdirectories when uploading
